# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

An [Upptime](https://upptime.js.org) instance (v1.41.3) that monitors externally-facing services on the operator's homelab and publishes a status page to `status.syntheticvisions.net` via GitHub Pages. Endpoints monitored are declared in `.upptimerc.yml` under `sites:` (currently Plex and Overseerr). Uptime checks run via GitHub Actions; history, response-time API JSON, and PNG graphs are committed back to the repo by the workflows themselves.

## The single source of truth: `.upptimerc.yml`

Almost everything user-editable lives in `.upptimerc.yml`:

- `sites:` — endpoint definitions (URL, expected status codes, header injection for Cloudflare Access, `__dangerous__body_down_if_text_missing` body assertions).
- `status-website:` — page metadata, intro copy, `cname`, `productionCacheTime` (localStorage cache for GitHub API responses — bumped to 30 min to soften the unauthenticated 60 req/hr/IP limit).
- `customHeadHtml` — Google Fonts preconnect + stylesheet for Space Grotesk / Geist / JetBrains Mono.
- `css` — the **entire SyntheticVisions brand layer**. It defines raw + semantic CSS variables, then overrides Upptime's Svelte-rendered SPA (`global.css` cascades after us, so most overrides need `!important`). Status-card border colors, tag pills, sticky nav blur, dot-grid background, and form controls are all driven here.

When changing the look-and-feel, edit the `css:` block in `.upptimerc.yml`, not the generated site output. Specificity notes are already inline in the CSS comments (e.g. `article.link.X` at 0,2,1 beats `article.X` at 0,1,1 regardless of source order — keep that in mind when adding rules).

## What you must NOT edit by hand

- **`.github/workflows/*.yml`** — generated and overwritten by Upptime's template-update workflow. Each file's header warns about this. Change `.upptimerc.yml` instead; `Setup CI` regenerates them.
- **`README.md` summary table** (between `<!--start: status pages-->` / `<!--end: status pages-->`) — written by `Summary CI`.
- **`README.md` "Live Status" line** at the top — written by `Summary CI`.
- **`history/*.yml`, `api/**/*.json`, `graphs/**/*.png`** — generated by `Uptime CI`, `Response Time CI`, and `Graphs CI` respectively. Treat as Open Database License data, not source.

If you need a workflow change that survives template updates, the upstream Upptime config (`workflowSchedule`, etc.) goes in `.upptimerc.yml`. If that's not enough, the change belongs upstream.

## The dispatcher (`dispatcher/`)

A Cloudflare Worker that fires every 5 minutes and POSTs `repository_dispatch` (`event_type: "uptime"`) to this repo so `uptime.yml` runs on a reliable cadence — GitHub's native `schedule:` cron is best-effort and drops triggers under load. Authenticated as a dedicated GitHub App (`homelab-status-dispatcher`) with `Repository contents: write`, installed only on this repo.

Deploy:

    cd dispatcher
    npm install
    npx wrangler secret put GITHUB_APP_PRIVATE_KEY   # paste PKCS#8 PEM
    npx wrangler deploy

Non-secret config is in `dispatcher/wrangler.toml` `[vars]`. Recovery procedures (key rotation, App uninstall) live in the operator's private homelab repo at `docs/runbooks/status-page.md` (bundle `0078-external-status-hosting`).

## Endpoint quirks

- **Plex** — the public endpoint sits behind no auth on `/identity` and returns 200 with `machineIdentifier` in the body; the `__dangerous__body_down_if_text_missing` assertion catches captive-portal / WAF interception that would otherwise look "up."
- **Overseerr** — gated by Cloudflare Access. The `CF-Access-Client-Id` / `CF-Access-Client-Secret` repo secrets are injected as request headers; `/api/v1/status` is the unauthenticated-once-past-Access health endpoint.

## Local preview harness

`@upptime/status-page` is a Sapper (Svelte) app with **no CLI** (`npm view @upptime/status-page bin` is empty). The only working local-dev path is to clone upstream and run it as a sibling of this repo's `.upptimerc.yml`, because `pre-process.ts` does `readFile(join("..", ".upptimerc.yml"))` relative to its cwd.

Canonical setup (see `~/Developer/homelab/docs/bundles/0080-homelab-status-redesign/tasks/task-01-local-preview-harness.md` for the full procedure and worklog):

```bash
mkdir -p /tmp/upptime-preview && cd /tmp/upptime-preview
git clone --depth 1 https://github.com/upptime/status-page.git
ln -sfn ~/Developer/homelab-status/.upptimerc.yml /tmp/upptime-preview/.upptimerc.yml

# Mirror prod's `cp -r ../assets/*` into Sapper's static dir per-file (so files
# under assets/ are served at URL root, no /assets/ prefix — same shape as prod).
mkdir -p /tmp/upptime-preview/status-page/static
for src in ~/Developer/homelab-status/assets/*; do
  ln -sfn "$src" /tmp/upptime-preview/status-page/static/"$(basename "$src")"
done

cd /tmp/upptime-preview/status-page
npm install
npm run dev   # Sapper binds to :3000
```

Open `http://Seans-Mac-mini.local:3000/` (LAN-reachable via mDNS, per `~/.claude/CLAUDE.md` § "Sharing HTML with the user") or `http://localhost:3000/`. Static-file edits under `assets/` are picked up immediately — no rebuild, just hard-refresh (Cmd+Shift+R) to bust the browser's CSS cache. Sapper rebuilds only trigger for `src/**` changes upstream, which this repo does not touch.

**The SPA fetches `api/`/`history/` data at runtime from `raw.githubusercontent.com/SynVisions/homelab-status/master/...`** — so local preview always reflects master-branch data, even on a feature branch. Fine for visual/CSS work; not useful for data-side debugging.

## Always verify CSS changes with Playwright before pushing

Manual eyeballing has missed at least two specificity bugs in this repo (status-card left-border disappearing, then reappearing 1px-mint instead of 4px-mint). Spec ties in this codebase are routine: Upptime's `global.css` and our inline `<style>` from `css:` both render in `<svelte:head>`, so source order and selector specificity decide every override. Don't trust the eye — run Playwright against `localhost:3000` and read `window.getComputedStyle(...)` for the property you're trying to change.

Cover the states that don't appear in current data by comparing against `https://demo.upptime.js.org/` (which exercises down / down-active / degraded / maintenance article variants) and/or by DOM-injecting the relevant classes onto an existing article in Playwright. See worklog `task-04-push-and-verify.md` rounds 3–4 for the pattern.

## Asset hosting (jsDelivr CDN, not root-relative)

`themeUrl`, `faviconSvg`, `logoUrl` in `.upptimerc.yml` are all pinned to `https://cdn.jsdelivr.net/gh/SynVisions/homelab-status@<sha>/assets/...`, **not** root-relative paths like `/theme.css`. Two reasons:

1. **Sapper's export crawler follows every `<link>`/`<src>` URL in the rendered HTML.** Root-relative paths to files that aren't in `status-page/static/` at export time cause Sapper to materialize `theme.css/index.html` (404) directories. The subsequent `cp -r ../assets/* status-page/__sapper__/export/` from `upptime/uptime-monitor` then fails `EISDIR` trying to write a file where a directory now exists. This broke CI on PR #3.
2. **jsDelivr's branch-ref URLs can be served with `Content-Type: text/plain`** on some edges, which trips `X-Content-Type-Options: nosniff` and silently blocks the stylesheet. Always pin to a commit SHA, never `@main` / `@master` / `@feature-branch`. SVG assets (logo, favicon) are not MIME-sniff-blocked and are safe on jsDelivr; CSS that needs to be bullet-proof should be inlined via the `css:` key instead of `themeUrl:` (which is what we currently do for the theme).

When changing brand assets, push first, then re-pin `.upptimerc.yml` to the new SHA in a follow-up commit. `Setup CI` re-deploys gh-pages on any push that touches `.upptimerc.yml`, on any branch (no branch guard) — `peaceiris/actions-gh-pages@v4` with `force_orphan: false` rewrites gh-pages from whichever branch pushed last.

## Conventions

- Times in user-facing prose: Pacific Time with offset (e.g. `2026-05-12 09:00 PDT`). Cron expressions, commit messages, and YAML history stay UTC.
- Commit messages from CI use the gitmoji style Upptime ships (`:bento: Update graphs`, `🟩 Service is up`, etc.); manual commits in this repo follow conventional-commits (`feat:`, `fix:`).
